# Source Translation Rules

## General Principles

1. **Never copy-paste** — Source text must be transformed, not transplanted
2. **Preserve key information** — Nothing critical gets lost; details move to speaker notes
3. **One idea per slide** — Regardless of source density
4. **Visual-first** — Ask "can this be shown instead of said?" for every piece of content
5. **Cite sources** — Always attribute source material

## Paper → Slides

### What to Extract
- **Core contribution**: 1-2 sentence summary of what's new
- **Key figures**: Often the most powerful slides — recreate or reference them
- **Methodology summary**: Simplified for the audience level
- **Key results**: 3-5 main findings with supporting data
- **Limitations**: Honest, brief acknowledgment
- **Conclusion**: Clear takeaway message

### Slide Structure
```
1. Title slide (paper title, authors, affiliation)
2. Motivation / Problem (why this research matters)
3. Background (only what's necessary for understanding)
4-5. Methodology (simplified, visual preferred)
6-10. Results (1 finding per slide, key figures)
11. Limitations & Future Work
12. Conclusion / Takeaway
13. References (if needed)
```

### Rules
- Extract key figures exactly — they're the paper's visual arguments
- Convert tables to charts when possible (charts read faster at projection)
- Statistical results: show effect size and confidence, not just p-values
- Move detailed methods to speaker notes
- Include paper DOI/citation in final slide

## Document → Slides

### What to Extract
- **Executive summary** or key conclusions
- **Main sections** with their core messages
- **Key data points**, statistics, or findings
- **Action items** or recommendations
- **Important quotes** or stakeholder statements

### Slide Structure
```
1. Title slide
2. Executive Summary / Key Takeaways (3-5 bullets max)
3-N. One section per major topic (1 key message per slide)
N+1. Action Items / Next Steps
N+2. Q&A / Discussion
```

### Rules
- Lead with conclusions, not background
- One metric per slide for data-heavy documents
- Convert tables to visual comparisons
- Keep original document structure where it makes sense
- Preserve exact numbers and attributions

## Outline → Slides

### What to Do
- **Validate structure**: Does the hierarchy flow logically?
- **Expand each point**: What visual or data would support it?
- **Identify gaps**: Where does the outline assume knowledge the audience may lack?
- **Suggest visuals**: For each slide, suggest image/diagram/chart type

### Rules
- Respect the author's intended hierarchy
- Top-level items → section breaks
- Second-level items → individual slides
- Third-level items → content within slides or speaker notes
- Flag points that are too vague to become slides

## Raw Text / Topic → Slides

### Process
1. **Research**: If given a topic, gather key facts and structure
2. **Identify themes**: Group related information
3. **Find narrative**: What story does this information tell?
4. **Structure**: Arrange into a logical flow with arc
5. **Design**: Suggest visuals for each section

### Rules
- Create a narrative arc: Hook → Context → Core → Evidence → Takeaway
- Don't just list facts — build an argument or story
- Research beyond the given text if needed for accuracy
- Every claim should be supportable
- Include sources for all factual claims

## Universal Rules

### Speaker Notes
- Every slide must have speaker notes
- Notes expand on the slide content (not repeat it)
- Notes include: talking points, transition phrases, timing cues
- Notes contain detail that was removed from the slide surface

### Visual Markers
- Mark needed visuals with: `<!-- TODO: [description of needed visual] -->`
- Suggest specific chart types for data: bar, line, scatter, etc.
- Suggest diagram types: flowchart, timeline, comparison, etc.
- Note when a photograph or illustration would be ideal

### Quality Criteria
- Does each slide pass the "billboard test"? (Readable in 3 seconds)
- Does the deck tell a complete story without speaker notes?
- Are all critical source details preserved somewhere (slide or notes)?
- Would the source author recognize their content in the slides?
