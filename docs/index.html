01

The Problem with Default Slides

Most decks fail for repeatable reasons. impeccable-quarto treats them as detectable anti-patterns, not vague matters of taste.

🧱
Major

Bullet Wall

Dense stacked bullets flatten hierarchy and force the audience to read instead of listen.

📦
Major

Content Dump

Whole paragraphs and raw notes land on one slide with no narrative filtering or visual structure.

🎨
Major

Generic Theme

Default templates create sameness, weak contrast choices, and no fit for the presentation context.

⚫
Major

Pure Black/White

Untinted extremes feel harsh on projection and ignore the nuance available in modern color spaces.

🔎
Major

Tiny Text

Slides break at room distance when type drops below practical readability thresholds.

📝
Major

Missing Notes

Without speaker notes, key context stays trapped in the presenter’s head instead of the deck system.

05

Get Started

Setup is intentionally lightweight. Clone the repo, render a deck, then preview and iterate against the scoring pipeline.

Step 1: Clone the repo

Pull down the source, themes, examples, and review rules in one pass.

git clone https://github.com/alohays/impeccable-quarto.git

Step 2: Create a deck

Render the demo deck to see the design system and scoring assumptions in action.

quarto render examples/demo.qmd

Step 3: Preview

Run a live preview while you adjust structure, notes, layout, and theme choices.

quarto preview your-deck.qmd
